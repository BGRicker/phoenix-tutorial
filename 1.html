Lesson 1

Getting Setup

The first step to building web applications with the Phoenix Web Framework is to install all the tools you'll need to get coding.

Installing Elixir

Follow the Install Instructions from the Elixir-Lang homepage. This will install Elixir and the Erlang programming language on your machine.

I used the homebrew instructions and it went through seemlessly. Once you have installed Elixir, you can verify it's properly installed by running the following commands: 

$ elixir --version
Elixir 1.0.2
$ erl -version
Erlang (SMP,ASYNC_THREADS,HIPE) (BEAM) emulator version 6.2

Elixir provides a REPL, similar to IRB, called iex. Jump into iex by running the following command: 

$ iex

As you would expect you can run standard Elixir code here: 

$ iex
Erlang/OTP 17 [erts-6.2] [source] [64-bit] [smp:4:4] [async-threads:10] [hipe] [kernel-poll:false] [dtrace]

Interactive Elixir (1.0.2) - press Ctrl+C to exit (type h() ENTER for help)
iex(1)> a = 10 + 14
24
iex(2)>

To get out of iex, press CTRL+C twice in a row.

When you get to the point where you need more of an rbenv type tool, checkout the exenv project.

The syntax to Elixir is very similar to ruby. The key differences are: macros instead of meta-programming, no classes or instance variables and pattern matching support.

Setting Up A Text Editor

Most text editors don't include syntax highlighting for Elixir source code by default. You can find packages for Sublime if you use that, but I'll walk you through how to add support for Elixir into atom. If you haven't installed the atom text editor, you can get it here.

Once you load up atom, in the menu click Atom>Preferences... Click the "packages" button, filter for the language-elixir package and install it. This will automatically highlight files with the extension exs as elixir source code.

Installing and Setting Up Postgres

Right now, Postgres is the only supported database by Ecto, which is what we'll be using to connect to our database. That makes the decision about what database to choose easy!

So if you don't have Postgres installed yet, Follow the Instructions to install it.

Then create an ecto user: 

$ createuser ecto --pwprompt

Then enter a password you'll remember.

Then create our database: 

$ createdb -Oecto -Eutf8 splurty_development

Then make sure you can log into your postgres database: 

$ psql splurty_development --user ecto --password

When you get pulled into the postgres database prompt, hold CTRL+D to exit.

Getting the Phoenix Framework

Go into the directory where you usually write code and clone the phoenix source code: 

$ git clone https://github.com/phoenixframework/phoenix.git

This tutorial is geared to work with Phoenix version 0.8. As it's changing quite quickly, you'll want to make sure you're using that version. 

$ git checkout v0.8.0

Starting your first project

You'll need to be in the same directory as the phoenix project you cloned when you run the command to create the first project: 

$ mix phoenix.new splurty ../splurty

This command will generate a new phoenix web project called splurty in the splurty directory of the parent folder.

This command is the first time we're using mix. Mix is a program that is a hybrid of bundler and rake if you're familiar with ruby on rails. In short it allows us to run commands and manages our dependencies.

If you've tried programming in scala, or other functional programming, you may have noticed package management usually tends to suck. Mix is very well designed and works seemlessly.

Next change the directory to the folder your splurty application is located in: 

$ cd ../splurty

Next compile your application's source code and also fetch and install all the dependencies: 

$ mix do deps.get, compile

It should complete without showing you any error messages and you can start your server by running the following command: 

$ mix phoenix.server

Then you can go to localhost:4000 and see your generated phoenix web application. Awesome!

Your application should look just like this at this step.
